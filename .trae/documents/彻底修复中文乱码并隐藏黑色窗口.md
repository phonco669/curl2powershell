## 问题分析
- 脚本中直接写入中文常量（如 GUI 文本与 `$query` 默认值）在 Windows PowerShell 读取脚本时可能被按本地 ANSI 编码解析，导致乱码；即使运行时设置 UTF-8 也无效，因为解析发生在执行前。
- `start.bat` 以控制台方式启动 PowerShell，必然出现黑色 `cmd.exe` 窗口；仅设置 `chcp` 或 `-WindowStyle Hidden` 会有闪烁。

## 解决方案
### 1. 中文资源外置并按 UTF-8 读取
- 新增 `tools/locale/zh.json`（UTF-8），包含：
  - `labels`: GUI 所有中文文案（标题、提示、按钮文字、状态提示）。
  - `defaults.query`: `$query` 的默认中文文案。
- 在 `tools/curl2ps.ps1` 中移除所有中文字面量，改为：
  - 启动时 `Get-Content -Encoding UTF8 | ConvertFrom-Json` 读取文案。
  - 使用读取到的字符串设置控件文本与生成代码中的 `$query` 默认值。
- 继续保留：`[Console]::Input/OutputEncoding = UTF8` 与 `-Encoding UTF8` 文件读取。

### 2. 隐藏黑色窗口
- 新增 `start.vbs` 使用 `WScript.Shell.Run` 以隐藏方式启动：
  - 命令：`powershell -NoProfile -ExecutionPolicy Bypass -File "tools\curl2ps.ps1"`
  - 第三个参数设为 `0`（隐藏窗口），确保无 `cmd.exe` 弹窗。
- 保留 `start.bat`（兼容命令行用户），但推荐用户双击 `start.vbs` 实现真正无黑窗启动。

## 代码改动点
- 添加 `tools/locale/zh.json`，示例键：
  - `title`, `pasteCurl`, `convert`, `copy`, `run`, `outputTitle`, `convertFail`, `runDone`, `runFail`, `psQueryDefault`。
- 更新 `tools/curl2ps.ps1`：
  - 顶部加载 JSON，所有控件的中文文本替换为资源变量。
  - 生成代码部分 `$query = '<中文默认>'` 改为从资源读取。
- 添加 `start.vbs`：隐藏启动 PowerShell GUI。

## 验证
- 直接运行 `start.vbs` 打开 GUI，确认：标题、标签、按钮、消息框中文正常显示，无黑色窗口。
- 运行命令行：`powershell -NoProfile -ExecutionPolicy Bypass -File tools\curl2ps.ps1 -InputFile tools\sample.curl`，检查生成的 `$query` 行中文是否正确。

## 影响
- 不改变现有业务逻辑与界面布局，仅调整文案来源与启动方式，确保中文可靠与体验更好。
